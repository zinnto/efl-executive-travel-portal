How data & publishing works

This is a static site (GitHub Pages) — there's no server or database behind it. Everything you add or edit in Trips and Executives is saved to this browser's local storage, which is why changes appear instantly when you preview a trip in the portal. To make a change permanent for everyone, download the trip's (or executive's) JSON, commit it into the right folder below, and push — GitHub Pages redeploys automatically.

Folder structure

data/
  trips-index.json          → directory of every trip
  executives-index.json       → directory of every executive
  trips/
    EFL-UG-0726.json             → one file per trip
    EFL-SG-0812.json
documents/                        → PDFs referenced by trips

Resolution order the portal uses

1. ?trip=<id> in the URL  →  2. last trip opened on that device  →  3. defaultTripId in trips-index.json.

Export & Import

Back up every trip and executive currently saved in this browser, or restore/seed from a backup file.

Danger Zone

Clears every trip and executive edit saved in this browser's local storage (does not touch files already committed to the repository). Use this if local preview data has gotten out of sync.
