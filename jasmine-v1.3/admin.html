Publish & Sync

Jasmine is a static site — data lives in this browser only, not on a server. Executives and trips added here won't be visible to anyone visiting the site in a different browser or device until you publish them.

Export Data downloads the current executives, trips, and settings as JSON files. Replace the matching files in data/ in the repo with these and redeploy to make this the data everyone sees on their first visit.

Reload Shipped Data discards whatever is currently saved in this browser and re-loads whatever is published in the live data/ files. Use this to pull down changes published from elsewhere.

GitHub Publishing

Publish executives, trips, and settings straight to your repo with one click, instead of exporting and committing manually. This also bumps the data version automatically, so your executives' browsers refresh to the new data as soon as they open their profile link — no action needed on their end.

Use a scoped token. Create a fine-grained personal access token limited to this one repository with Contents: Read and write permission only — not a token with full account access. It's stored in this browser only and is never included in data exports. Treat it like a password: only save it on a device you trust, and revoke it from GitHub if that ever changes.
